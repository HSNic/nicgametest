# 加掛版更新 SOP

建立日期:2026-07-06

## 適用情境

當原作者或上游加掛版有新版本,需要把新版合併到本專案時使用。

## 核心原則

- 原作者本體用「同步」,不要手改。
- 本專案功能用 `afk-*.js` 外掛保留。
- 同步後一定要跑驗證,不能只看首頁能開。
- 有任何會影響玩家存檔的改動,必須額外檢查 localStorage 與匯入匯出。

## 更新前檢查

| 步驟 | 指令或檢查 | 目的 |
|---|---|---|
| 1 | `git status --short --branch` | 確認工作樹是否乾淨 |
| 2 | 備份目前可玩的版本 | 避免更新失敗無法回復 |
| 3 | 記錄 `version.json` | 確認目前版本 |
| 4 | 記錄 `last-sync.json` | 確認上次同步時間 |
| 5 | 檢查 `index.html` script 順序 | 確認外掛載入在原作者 `js/*.js` 之後 |

## 建議更新流程

### 1. 先建立更新分支

```bash
git switch -c update-upstream-YYYYMMDD
```

### 2. 執行同步腳本

```bash
node scripts/sync-upstream.mjs
```

同步腳本目前會做這些事:

| 項目 | 說明 |
|---|---|
| 抓原作者首頁 | 來源為 `https://shines871.github.io/idle-lineage-class/index.html` |
| 重抓 `js/*.js`、`css/*.css` | 依上游 `index.html` 的引用自動抓取 |
| 改寫 `?v=` | 用內容 sha1 破快取 |
| 掛回外掛 | 依 `PLUGINS` 陣列把 `afk-*.js` 插回 `</body>` 前 |
| 補靜態資源 | 比對 `assets/`、`public/` |
| 重建 manifest | 更新 `assets-manifest.json`、`anim-manifest.json` |
| 更新同步時間 | 寫入 `last-sync.json` |

### 3. 檢查本次差異

```bash
git diff --stat
git diff -- index.html scripts/sync-upstream.mjs sw.js version.json last-sync.json
```

需要特別看:

| 檔案 | 檢查重點 |
|---|---|
| `index.html` | `afk-*.js` 是否仍全部存在、順序是否正確 |
| `js/*.js` | 原作者是否新增或刪除檔案 |
| `css/*.css` | 是否有新樣式檔 |
| `assets-manifest.json` | 是否因大量圖片變動而更新 |
| `anim-manifest.json` | 怪物動畫是否有大量變動 |
| `last-sync.json` | 是否更新為本次時間 |

### 4. 跑 smoke test

```bash
node scripts/smoke-hooks.mjs
```

若失敗,優先判斷:

| 失敗類型 | 可能原因 |
|---|---|
| 外掛 hooks 掛不上 | 原作者 DOM id 或全域函式改名 |
| 子資源 404 | `js/`、`css/`、`assets/`、`public/` 漏同步 |
| 地圖名英文外露 | `AFK_EXTRA.mapName` 沒跟上新地圖 |
| JS error | 外掛依賴的全域資料結構改變 |

### 5. 手動功能驗收

| 功能 | 桌機 | 手機 |
|---|---|---|
| 首頁可進入 | 必測 | 必測 |
| 建立角色 / 載入角色 | 必測 | 必測 |
| 戰鬥 tick 正常 | 必測 | 必測 |
| 離線掛機結算 | 必測 | 建議測 |
| 存檔匯出 / 匯入 | 必測 | 必測 |
| 怪物 / 掉落查詢 | 必測 | 必測 |
| 小百科搜尋 | 必測 | 必測 |
| PWA / Service Worker | 建議測 | 必測 |
| 木人場 | 建議測 | 建議測 |

### 6. 更新版本與快取

若修改了任何 `afk-*.js`,要確認:

| 項目 | 說明 |
|---|---|
| `index.html` | 對應 `<script src="afk-xxx.js?v=...">` 已 bump |
| `sw.js` | `CODE_VERSION` 有更新 |
| `version.json` | `code`、`build` 有更新 |
| `assets-manifest.json` | 圖片快取對帳正常 |
| `anim-manifest.json` | 動畫快取對帳正常 |

### 7. Commit

建議拆成兩種 commit:

| Commit | 內容 |
|---|---|
| `sync upstream YYYY-MM-DD` | 純同步原作者本體與 manifest |
| `fix plugin after upstream sync` | 修外掛掛點或資料對應 |

## 發布前檢查清單

- `git status` 只有預期檔案。
- `index.html` 中所有 `afk-*.js` 都存在於本機。
- `index.html` 中所有 `js/*.js`、`css/*.css` 都存在於本機。
- 首頁背景圖沒有 404。
- 手機首頁沒有版型爆掉。
- 小百科與掉落查詢沒有英文 key 外露。
- 未載入角色時,不會呼叫原作者 `saveGame()` 覆蓋存檔。
- Service Worker 版本已更新。
- 至少測過一個舊存檔載入與匯出。

## 不建議做的事

- 不要直接改 `js/*.js` 來修外掛需求。
- 不要把玩家 localStorage 當成可任意重寫的資料庫。
- 不要只改 README 或畫面文字,卻沒檢查實際計算邏輯。
- 不要在沒有 diff 與 smoke test 的情況下發布。
