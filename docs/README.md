# 接手維護文件索引

建立日期:2026-07-06

本資料夾整理接手 `idle-lineage-class` 加掛版時最需要的分析文件。目標是讓後續修改、同步上游、驗證發布都有固定依據,避免每次靠人工記憶。

## 文件清單

| 文件 | 用途 |
|---|---|
| `接手分析_20260706.md` | 專案架構、檔案責任、目前基準、建議維護策略 |
| `更新SOP_20260706.md` | 從上游同步、比對、驗證、發布前檢查流程 |
| `風險清單_20260706.md` | 目前最容易壞的地方與檢查方式 |
| `外掛依賴矩陣_20260706.csv` | 每支 `afk-*.js` 的功能、依賴、存檔風險、更新風險 |

## 使用順序

1. 先讀 `接手分析_20260706.md`,理解「原作者本體」與「加掛外掛」的邊界。
2. 修改功能前,查 `外掛依賴矩陣_20260706.csv`,確認該外掛是否會碰 DOM、全域函式或 localStorage。
3. 跟上游更新前,照 `更新SOP_20260706.md` 做。
4. 驗證時,對照 `風險清單_20260706.md` 逐項檢查。

## 核心原則

- 不直接修改原作者本體:`index.html`、`js/*.js`、`css/*.css`。
- 自訂功能放在 `afk-*.js`。
- 更新上游時,先重抓原作者本體,再把外掛 script 掛回。
- 任何會寫入玩家存檔的操作都要特別審查。
