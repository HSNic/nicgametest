# 維護風險清單

建立日期:2026-07-06

## 高風險

| 風險 | 影響 | 偵測方式 | 建議處理 |
|---|---|---|---|
| 未載入角色時呼叫 `saveGame()` | 可能把真實存檔覆蓋成空白角色 | 搜尋 `saveGame`、測主選單匯出/返回/關頁 | 保留 `afk-fixes.js` 的空白角色防呆 |
| 原作者 DOM id 改名 | 外掛按鈕、modal、手機版掛不上 | `scripts/smoke-hooks.mjs`、瀏覽器 console | 修外掛 selector,不要改原作者 DOM |
| 原作者全域函式改名 | monkey-patch 失效,功能悄悄退化 | 搜尋 `window.xxx = function` 包裝點 | 外掛安裝時檢查函式存在,缺少時 console warn |
| 掉落表或製作表新增 | 掉落查詢、小百科漏資料 | grep `_DROPS`、`_RECIPES`、`gainItem` | 補 `afk-dex.js`、`afk-wiki.js` 的索引來源 |
| Service Worker 快取舊檔 | 玩家讀到舊 JS/CSS/圖片 | 檢查 `?v=`、`sw.js CODE_VERSION`、manifest | 修改外掛後 bump 版本並 stamp |

## 中風險

| 風險 | 影響 | 偵測方式 | 建議處理 |
|---|---|---|---|
| 新增 `public/assets/` 圖片漏同步 | 首頁或登入圖破圖 | Network 404、黑背景 | 確認同步腳本涵蓋 `assets/` 與 `public/` |
| 外掛載入順序錯誤 | 資料補充讀不到、API 尚未定義 | 檢查 `index.html` 後段 script 順序 | 保持 `afk-extradata.js` 在 dex/wiki 前 |
| 手機版 CSS 被上游改動影響 | 手機版跑版或按鈕重疊 | 手機 viewport 實測 | 改 `afk-mobile.js` 或外掛 CSS |
| PWA 安裝與更新流程失效 | 使用者裝置離線版不更新 | standalone 模式測試 | 測 `afk-pwa.js` 與 `sw.js` |
| 小百科英文 key 外露 | 使用者看到內部 id | 搜尋畫面是否出現英文字母 key | 補 `AFK_EXTRA.mapName` 或翻譯對照 |

## 低風險但常見

| 風險 | 影響 | 偵測方式 | 建議處理 |
|---|---|---|---|
| README 與實際功能不同步 | 接手者誤判 | 對照 `index.html` 的外掛清單 | 文件更新時同步 README 或 docs |
| `last-sync.json` 沒更新 | 不知道同步時間 | 讀 JSON | 同步腳本成功後確認 |
| `version.json` 沒更新 | 玩家端更新提示不準 | 讀 JSON | 修改程式後更新版本 |
| 大量圖片 manifest 變動 | commit 很大、review 困難 | `git diff --stat` | 單獨 commit manifest/資源同步 |

## 存檔安全檢查

凡是動到以下項目,都要特別審查:

| 關鍵字 | 風險 |
|---|---|
| `saveGame` | 可能寫入玩家目前 slot |
| `loadGame` | 可能觸發離線結算或 UI 還原 |
| `localStorage.setItem` | 可能改玩家資料 |
| `lineage_idle_save_` | 直接碰角色存檔 |
| `currentSlot` | 可能不是使用者以為的 slot |
| `player` | 未載入角色時可能是空白預設 |
