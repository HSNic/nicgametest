# 版本異動紀錄

這份文件記錄每次改動「改了什麼、為什麼改」，給使用者(非工程背景)快速看懂,
也保留未來要在遊戲畫面上顯示更新日誌時可以直接引用的素材。

**維護方式(往後每次改動都照此格式新增一筆,寫在最上面、新的在前):**

```
## <日期> <版本標籤(對應 version.json 的 build)>

**類型:** 小版本(外掛/樣式調整) 或 大版本(同步原作者新版)

**這次改了什麼(給玩家看的白話版):**
- 條列,每項一句話講清楚玩家會感受到的變化

**技術細節(給之後維護者看):**
- 對應的 commit / 改了哪些檔案 / 為什麼這樣做

**測試結果:**
- smoke test 結果、手動測試結果
```
