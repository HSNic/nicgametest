```

- 條列

**測試結果:**
- smoke test 結果、手動測試結果
```

## 2026-07-07 build 0707-2046(commit `7cf20aba`)

**類型:** 小版本(三項 UI 優化)

**改了哪些檔案、為什麼:**
- 依 `Lineage/待辦-ClaudeCode/2026-07-07_三項UI優化(道具分類_倉庫搜尋_自動販賣bug).md`,核對 v3.0.114 程式碼位置準確後,全部以外掛(monkey-patch)方式實作,未動 `js/10-ui-tabs.js`、`js/12-npc-quests.js` 本體。
- 新增 `afk-autosell-fix.js`:monkey-patch `setAutoSellOverride`/`deleteAutoSellOverride`,呼叫原函式前先暫存 `#as-item-search`/`#as-item-type`/`#as-item-scope`/`.as-box` 的 `scrollTop`,原函式整個重建 modal 後用 `setTimeout(0)` 回填,修復點「永遠保留/永遠販賣」後這些狀態被重置的 bug。
- 新增 `afk-warehouse-search.js`:monkey-patch `renderWarehouseNPC`,每次重繪後插入 `<input type="search">`,用模組變數 `query` 記憶關鍵字,對 `#wh-inv-list`/`#wh-store-list` 內帶 `data-tip-uid` 的項目做文字比對顯示/隱藏。
- 新增 `afk-item-subfilter.js`:monkey-patch `renderTabs`,在 `tab-items`/`tab-weapons`/`tab-armors` 插入子分類按鈕列。重用 `whItemSubCat()`(純函式)與 `EQUIP_CATEGORIES`/`equipCatKey()` 判斷子分類,**不**共用倉庫自己的 `_whFilter`/`_whSubFilter` 模組狀態(另開 `_subFilterState`)。物品列與 DOM `.list-item` 的對應靠「渲染順序」比照原函式 `player.inv.forEach` 依 `type` 分流的規則重建,沒有 `data-id` 屬性可直接讀。
  - **修復一個實測發現的 bug**:`afk-classic-list.js` 對 `.list-item` 下了 `display:flex!important`,`applyOne()` 原本用不帶 `!important` 的 `style.display='none'` 隱藏物品列,優先權打不贏,導致篩選按鈕會反應(active class 變色)但清單視覺上沒有真的篩選。改用 `style.setProperty('display','none','important')` 隱藏、`removeProperty('display')` 顯示(交回原本的 `!important` 規則決定顯示方式)才生效。用 Playwright 讀 `getComputedStyle`/`offsetParent` 確認修好後畫面真的有篩選,不是只看行內樣式字串。
- `index.html` 掛載 3 支新外掛;`scripts/sync-upstream.mjs` 的 `PLUGINS`、`scripts/smoke-hooks.mjs` 的 `need` 都已補上對應項目;`docs/外掛依賴矩陣_20260706.csv` 補三列。`sw.js`/`version.json` 由 `stamp-sw-version.mjs` 自動更新。

**測試結果:**
- `node scripts/smoke-hooks.mjs`:18 支外掛全過。
- Playwright 自動化測試:①子分類篩選(點「卡片」後只留卡片類可見,`getComputedStyle` 確認真的 `display:none`)、②倉庫搜尋(篩選正確、重繪後關鍵字不清空)、③自動販賣修復(觸發後搜尋字/捲動位置正確保留)均通過。
- 使用者實機測試(桌機瀏覽器)三項功能皆確認正常;過程中發現並修好上述 `!important` bug。附帶排查一個跟本次改動無關的既有問題:角色裝備「影子長靴」缺少 `assets/anim/影子長靴/` 動畫圖檔(比對同步前備份,此缺口為原本就存在,非本次改動造成,純視覺、不影響戰鬥數值)。

## 2026-07-07 build 0707-2001(commit `84262675`)

**類型:** 小版本(套用外部外掛更新 + 樣式調整)

**改了哪些檔案、為什麼:**
- 來源:`加掛版/原加掛版作者新版` 資料夾——注意這是**加掛版外掛套件原作者**的更新,不是遊戲本體原作者秋玥的更新,兩者是不同上游(該資料夾 `GAME_VERSION` 仍是 v3.0.65,比我們目前同步到的 v3.0.114 舊,因此**沒有**跑 `sync-local-upstream.mjs` 整份同步,只挑套用外掛檔案本身)。

- `afk-syncinfo.js`:改用新版結構(首頁顯示版本號而非同步時間),但**修改**了原始檔案兩處:拿掉「本專案自 2026-07 起不再跟進原作者版本」的註記(不適用,我們持續同步),版本來源改讀本專案既有的 `version.json` 的 `build` 欄位(原版讀 `app` 欄位,需要額外的 `/release` 發版腳本,我們沒有)。
- `afk-skin.js`:`MARQUEE_TEXT` 原本是固定字串「伺服器永久開放，但不再跟進原作者版本」,改成 `updateMarqueeVersion()` 非同步 fetch `version.json`,把跑馬燈兩個 `.afk-mq-seg` 文字改成「加掛版 build `<build>`」;fetch 失敗或 `file://` 協定則維持預設文字「加掛版」。
- `README.md`:使用者自行編輯,更新原作者/加掛版本歸屬(加掛版原作者為巴哈 Chaos)與非商業聲明。
- `index.html` bump 對應 4 支外掛 + `afk-skin.js` 版本號;`sw.js`/`version.json` 由 `stamp-sw-version.mjs` 自動更新。

**測試結果:**
- `node scripts/smoke-hooks.mjs`:15 支外掛全過。
- Playwright 自動化檢查:頁面載入無新增 console error(僅既有音效探測 404,非本次造成);跑馬燈文字確認渲染成「加掛版 build 0707-2001」。
- 使用者實機測試待補(離線結算數值、手機發熱、自動購買)。
