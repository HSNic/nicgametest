### 0. 切換到工作資料夾


### 1. 發布前檢查（`加掛版/docs/更新SOP_20260706.md`）
- `git status` 確認只有預期檔案有改動
- `index.html` 中所有 `afk-*.js`、`js/*.js`、`css/*.css` 都存在
- 跑 `node scripts/smoke-hooks.mjs`
- 手動測：首頁進入、建角/載入、戰鬥 tick、存檔匯出匯入
- 確認 `sw.js` 的 `CODE_VERSION`、`version.json` 已 bump

### 2. 更新 README 網址並 commit
把 `README.md` 第 7 行改成：
```
https://hsnic.github.io/nicgametest/
```
然後：
```bash
git add README.md
git commit -m "docs: 更新上線網址為 nicgametest"
```

### 3. 接上 GitHub remote 並 push（第一次才需要）
```bash
git remote add origin https://github.com/HSNic/nicgametest.git
git branch -M main
git push -u origin main
```
如果 push 要求輸入密碼且失敗，代表 GitHub 已不接受帳密登入，需要改用 Personal Access Token 或 SSH key（GitHub 網頁會提示怎麼做）。

### 4. 開啟 GitHub Pages
1. 到 repo 頁面 → `Settings` → 左側 `Pages`
2. `Source` 選 `Deploy from a branch`
3. Branch 選 `main`，資料夾選 `/ (root)` → `Save`
4. 等 1–2 分鐘，網址會顯示在同一頁（就是 `https://hsnic.github.io/nicgametest/`）

### 5. 驗收
- 用無痕視窗開啟 `https://hsnic.github.io/nicgametest/`，確認能進首頁、建角/載入正常（首頁已設 no-cache，不用擔心舊快取）
- 之後每次更新只要重複「發布前檢查 → commit → push」，Pages 會自動重建，不用再重設 remote/Pages
